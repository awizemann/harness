Not a UI test. A user test.

No accessibility identifiers. No source-code access. No pre-written plan. The agent reasons from "what's on screen" — the way a person would.

  1. Write the goal

    "I want to sign up and create my first list." "Delete my account." "Find a vegetarian restaurant near me and save it."

  2. Pick a persona

    "First-time user, never seen the app." "Returning power user." "Person in a hurry on a flaky network."

  3. Watch it run

    Harness builds your project, boots the simulator, and the agent drives. Every screen and action is logged.

Three artifacts come out of every run.

Did the goal complete?

A verdict — success, failure, or blocked — with a one-paragraph summary in the agent's own words.

What was the path?

A replayable timeline of every screen the agent saw and every action it took. Scrub it like a video.

Where was the friction?

Timestamped events the agent flagged as confusing, ambiguous, or dead-end — typed by friction kind so you can scan them.

How the agent works

A loop of see → reason → act, powered by Claude tool use. Each cycle: take a screenshot, give it to the model with the goal and recent history, get back a tool call (tap, swipe, type, scroll, screenshot, mark friction, finish). Append to the run log. Repeat.

  • Vision-grounded. Coordinates come from the model's read of the screenshot, not from a11y trees or stable selectors.
  • Persona-conditioned. The same goal under "first-time user" vs. "returning expert" produces different paths and different friction.
  • Replayable. Every run writes append-only JSONL plus screenshots. Open an old run and step through it like a flight recorder.
  • Cycle-aware. A built-in detector spots loops and forces the agent to break out or call it.

Deeper details on Agent-Loop, Tool-Schema, and Run-Replay-Format on the wiki.

Get started

Harness is alpha. Build from source today; binary releases follow.

Prerequisites

  • macOS 14 or later, Xcode 16+ (Swift 6 strict concurrency)
  • Homebrew + idb-companion + xcodegen
  • An Anthropic API key (stored in the macOS Keychain on first run)

Build

git clone https://github.com/awizemann/harness.git
cd harness
git submodule update --init --recursive
brew tap facebook/fb && brew install idb-companion xcodegen
xcodegen generate
open Harness.xcodeproj

Full setup, including first-run WDA build details, lives on Build-and-Run.

Contributing

PRs welcome. The contributing guide covers setup, the architecture rules, and the public-surfaces sync rule. Each public surface (this site, the wiki, the README) is updated in the same PR as the code change that affects it.

  • Architecture: MVVM-F, Swift 6 strict concurrency, single subprocess actor.
  • Standards live in standards/ — read them before adding code.
